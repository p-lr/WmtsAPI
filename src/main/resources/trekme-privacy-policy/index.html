Device permissions for Personal Data access

Depending on the User's specific device, this Application may request certain permissions that allow it to access the User's device Data as described below.

By default, these permissions must be granted by the User before the respective information can be accessed. Once the permission has been given, it can be revoked by the User at any time. In order to revoke these permissions, Users may refer to the device settings or contact the Owner for support at the contact details provided in the present document.
The exact procedure for controlling app permissions may be dependent on the User's device and software.

Please note that the revoking of such permissions might impact the proper functioning of this Application.

If User grants any of the permissions listed below, the respective Personal Data may be processed (i.e accessed to, modified or removed) by this Application.

Precise location permission

Used to show the position of the User on a map. The collected location data is used on the User's device only and is never sent to any third parties through the network.

Background location permission

Used to record the location of the User even when the application is closed or in the background. Such a recording is started by the User only, and a foreground service explicitly indicates that a recording is running. The collected location data is used on the User's device only and is never sent to any third parties through the network.

Storage permission

Used for accessing the file system and store Application Data. This permission is asked on Android 9 and below.

Wifi state permission

Allows map sharing with other Users using Wifi direct protocol.

Detailed information on the processing of Personal Data

Personal Data is collected for the following purposes and using the following services:

  • User positioning

    TrekMe is an Application made for hiking, biking, and other outdoor activities where the User wants to know his position on a map. Precise location is required in such circumstances.
    This service isn't activated automatically, but requires explicit authorization by the User.

    Retention of the location Data

    This service uses the location data to update relevant visual components of the Application, but does not store this Data. Once the Application is closed, the location data is lost.

  • GPX recording

    TrekMe is an Application made for hiking, biking, and other outdoor activities. The User might find useful to record his(her) location during a hike, for later usage and analysis.
    This service is only activated on an explicit User action from inside the Application, and requires an explicit authorization by the User.
    A prominent dialog tells the User that the location will be recorded even if the Application is closed or in the background.
    This dialog is shown before asking the User the grant the relevant permission.

    Retention of the location Data

    This service uses the location data to write GPX compliant files on the User's device. The recorded location data is never sent to any third parties through the network. GPX files remains on the User's device until the User manually removes those files or uninstalls the Application.

  • Storage permission

    This Application requests storage permission on Android 9 and below. This Application uses the User's device internal storage to store Application data such as map images. This service requires an explicit authorization by the User.
    At installation, the Application creates a folder named "trekme" in the internal memory and all Application data is stored in this folder. The Application never attempts to read or write Data outside of this folder.
